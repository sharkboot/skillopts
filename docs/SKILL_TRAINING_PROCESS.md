# SKILL.md Training Process Documentation
# SpreadsheetBench with DashScope qwen3.6-35b-a3b

## Overview

This document records the complete process of how an empty SKILL.md is progressively 
trained and optimized for SpreadsheetBench using the SkillOpt framework with the 
DashScope API and qwen3.6-35b-a3b model.

## Training Configuration

- **Model**: qwen3.6-35b-a3b
- **API**: DashScope (https://dashscope.aliyuncs.com/compatible-mode/v1)
- **Training Samples**: 10
- **Validation Samples**: 3
- **Training Epochs**: 3
- **Update Mode**: patch

## Step-by-Step Training Process

### Step 1: Initial State (Empty SKILL.md)

The initial SKILL.md starts as an empty template:

```markdown
# Spreadsheet Manipulation Skill (Empty Initial)

This skill file is intentionally left empty for training optimization.
The optimizer will generate appropriate instructions based on training feedback.
```

### Step 2: Initial Evaluation with Empty Skill

Before any optimization, the baseline performance is measured by running the model
with the empty SKILL.md on the training and validation sets.

**Baseline Results**:
- Expected: Low performance due to lack of domain-specific guidance
- Purpose: Establish a baseline for measuring improvement

### Step 3: First Training Iteration

1. **Rollout Phase**: Model generates code for each training sample
2. **Evaluation Phase**: Generated code is executed and compared against expected outputs
3. **Analysis Phase**: Failed cases are analyzed to identify common failure patterns
4. **Optimization Phase**: SKILL.md is updated with new instructions based on analysis

**SKILL.md After Iteration 1**:
(Will be generated by the optimizer based on training feedback)

### Step 4: Subsequent Iterations

Each iteration follows the same pattern:
1. Evaluate current SKILL.md on training samples
2. Analyze failures and successes
3. Update SKILL.md to improve performance
4. Track improvements in validation score

### Step 5: Final Optimized SKILL.md

The final SKILL.md contains all learned patterns and best practices for 
spreadsheet manipulation tasks.

## Training Loop Details

### Rollout Phase
For each training sample:
1. Load input spreadsheet
2. Apply instruction using the current SKILL.md
3. Generate Python code
4. Execute code
5. Compare output with expected result
6. Record trajectory

### Analysis Phase
Analyze trajectories to identify:
- Common error patterns
- Successful strategies
- Missing knowledge areas

### Optimization Phase
Update SKILL.md using:
- Patch-based updates for targeted fixes
- Full rewrites for comprehensive improvements
- Meta-skill guidance for high-level strategies

## Usage

To reproduce this training:

```bash
# Set environment
export QWEN_CHAT_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
export QWEN_CHAT_API_KEY=your_api_key

# Run training
python scripts/run_training.py

# Or use the config directly
python scripts/train.py --config configs/spreadsheetbench/dashscope_qwen36.yaml
```

## Output Files

Training outputs are saved to:
- `outputs/skillopt_spreadsheetbench_qwen36_<timestamp>/`

Key files:
- `skill.md`: Current SKILL.md state
- `results.jsonl`: Detailed evaluation results
- `trajectories/`: Full training trajectories
